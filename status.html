Description: Checks the health status of the API, providing information about its operational state, version, cache usage, and uptime.

Use Case: Use this endpoint to verify if the API is running correctly before making other requests. Ideal for monitoring and debugging.

Parameters: None

Example Request:

GET /health

Example Response:

{
  "status": "healthy",
  "timestamp": "2025-07-18T08:26:00.123456+06:00",
  "version": "2.0.0",
  "cache_size": 5,
  "uptime": 1234567.89
}

Response Fields:

  • status: Indicates if the API is "healthy" or not.
  • timestamp: ISO 8601 timestamp of the response.
  • version: API version (e.g., 2.0.0).
  • cache_size: Number of cached responses.
  • uptime: Time (in seconds) since the API started.

Notes:

  • Call this endpoint periodically to ensure the API is responsive.
  • A non-200 status code or "status" not equal to "healthy" indicates an issue.

Description: Fetches P2P trading data from Binance, including seller details, prices, and payment methods for specified assets and fiat currencies.

Use Case: Retrieve real-time P2P offers for trading cryptocurrencies (e.g., USDT) with fiat (e.g., BDT) using specific payment methods (e.g., bKash).

Parameters:

Parameter Type Description Default
asset String Cryptocurrency (e.g., USDT, BTC, ETH). Supported: USDT, BTC, ETH, BNB, BUSD, ADA, DOT, MATIC, SHIB, DOGE. USDT
pay_type String Fiat currency (e.g., BDT, USD). Supported: BHD, AED, SAR, EGP, JOD, KWD, LBP, MAD, OMR, QAR, TND, BDT, INR, PKR, USD, EUR, GBP, TRY, RUB, NGN, KES, ZAR, PHP, THB, MYR, SGD, HKD, JPY, KRW, CNY, VND, IDR, BRL, ARS, CLP, COP, MXN, PEN, GHS, UGX, TZS, LRD, SLL, GMD, MRO, CVE. BDT
pay_method String Payment method (e.g., BKASH, ALL). Depends on pay_type (e.g., BDT: BKASH, NAGAD, ROCKET, UPAY, BANK). ALL
trade_type String Trade direction: BUY or SELL. SELL
limit Integer Number of results to return (1-500). 100
sort_by String Sort results by: price, completion_rate, available_amount, monthly_orders. price
order String Sort order: asc or desc. asc
min_completion_rate Float Minimum seller completion rate (0-100). None
min_orders Integer Minimum seller monthly orders. None
online_only Boolean Show only online merchants (true/false). false

Example Request:

GET /api/v1/p2p?asset=USDT&pay_type=BDT&pay_method=BKASH&trade_type=SELL&limit=100

Example Response:

}

Response Fields:

  • success: Boolean indicating request success.
  • data: Array of seller objects with details (id, seller_name, price, etc.).
  • count: Number of sellers returned.
  • total_found: Total sellers matching criteria.
  • total_sellers: Same as total_found.
  • time_taken: Time (seconds) to process the request.
  • trade_type: Requested trade type (BUY/SELL).
  • api_dev: Developer handle (@ISmartCoder).
  • updates_channel: Telegram channel for updates.
  • statistics: Aggregated stats (avg_price, min_price, max_price, total_available).
  • parameters: Echo of request parameters.
  • timestamp: ISO 8601 timestamp.
  • cache_status: Indicates if response was cached.

Notes:

  • Use limit to control response size (max 500).
  • Responses are cached for 30 seconds to improve performance.
  • Invalid parameters return a 400 error with details.
  • Try the test form above to experiment with parameters.

Description: Retrieves the list of supported payment methods for each fiat currency.

Use Case: Use this to populate payment method options in your application based on the selected fiat currency.

Parameters: None

Example Request:

GET /api/v1/p2p/methods

Example Response:

{
  "success": true,
  "data": {
    "BHD": ["BANK", "BBK", "DENIZBANK", "AIC", "CASHU", "MONEYGRAM"],
    "BDT": ["BKASH", "NAGAD", "ROCKET", "UPAY", "BANK"],
    "USD": ["WISE", "PAYPAL", "BANK", "ZELLE"],
    ...
  },
  "timestamp": "2025-07-18T08:26:00.123456+06:00"
}

Response Fields:

  • success: Boolean indicating request success.
  • data: Object mapping fiat currencies to their payment methods.
  • timestamp: ISO 8601 timestamp.

Notes:

  • Each fiat currency has a unique set of payment methods.
  • Use this endpoint to validate pay_method in P2P requests.
